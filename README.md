
# 苹果品质分析系统

## 项目简介

苹果品质分析系统是一个集 **数据展示、数据可视化、机器学习建模、品质预测、大数据处理、实时消息处理、AI智能报告生成** 于一体的综合性分析平台。

系统围绕苹果品质预测这一核心任务展开，通过分析苹果的多个特征指标，如：

- 大小（size）
- 重量（weight）
- 甜度（sweetness）
- 脆度（crunchiness）
- 多汁程度（juiciness）
- 成熟度（ripeness）
- 酸度（acidity）

对苹果品质进行分类预测，并提供完整的数据分析与展示能力。

在传统 Flask + 机器学习项目基础上，本系统进一步扩展了：

- **Spark + Hive**：用于大数据读取与处理
- **Kafka**：用于数据变更消息传递与实时处理
- **AI 报告生成**：用于自动生成自然语言分析报告

因此，本项目不仅适合作为机器学习课程项目，也适合作为大数据综合实训项目进行展示与答辩。

---

## 项目功能

### 1. 数据展示
- 从 Hive 数据库读取苹果品质数据
- 展示原始数据内容
- 支持分页浏览
- 查看数据基本统计信息
- 检测数据是否发生更新

### 2. 数据可视化
- 品质分布图
- 特征分布图
- 相关性热力图
- 雷达图
- 多维特征对比分析

### 3. 数据处理
- 缺失值处理
- 重复值处理
- 数据类型转换
- 标签标准化
- 异常值处理
- 特征标准化
- 类别平衡处理

### 4. 模型训练
系统支持多种机器学习算法：
- 高斯朴素贝叶斯
- BP 神经网络
- 逻辑回归
- 随机森林
- K近邻
- Bagging

并支持：
- 模型训练
- 模型评估
- 准确率对比
- 模型保存

### 5. 品质预测
- 用户输入苹果特征值
- 选择指定模型
- 输出预测结果（优质 / 劣质）

### 6. Spark + Hive 大数据处理
- 使用 Spark SQL 查询 Hive 数据
- 支持分布式数据处理
- 支持大规模数据读取与统计分析
- 为可视化与模型训练提供数据支持

### 7. Kafka 实时处理
- 数据新增时发送消息
- 数据修改时发送消息
- 数据删除时发送消息
- 消费端处理变更数据
- 前端可感知数据更新状态

### 8. AI 智能报告生成
- 配置 AI 模型信息
- 测试 AI 接口连接
- 自动生成数据分析报告
- 支持流式输出
- 增强系统智能化展示效果

├── 应用层

│   └── 业务逻辑处理
│
├── 数据处理层
│   ├── Pandas数据清洗
│   ├── Spark数据处理
│   ├── 特征工程
│   └── 模型训练与预测
│
├── 实时消息层
│   ├── Kafka生产者

│
├── AI智能层

│

    ├── Hive数据表

    └── 配置文件

## 核心流程

### 1. 数据分析与预测流程

```text
Hive数据 / 原始数据
        ↓
数据读取
        ↓
数据清洗与预处理
        ↓
特征分析与可视化
        ↓
模型训练与评估
        ↓
品质预测
        ↓
AI生成分析报告
```

### 2. 实时处理流程

```text
用户执行增删改操作
        ↓
更新数据表
        ↓
发送Kafka消息
        ↓
Kafka消费者接收并处理
        ↓
记录数据变更日志
        ↓
前端检查更新状态
        ↓
页面刷新显示最新数据
```

## 安装与运行

## 1. 获取项目

如果是克隆仓库：

```bash
git clone https://github.com/14linxi/Apple-Quality-Analysis.git
cd Apple-Quality-Analysis
```

如果是本地已有项目，则直接进入项目目录即可。

---

## 2. 安装 Python 依赖

建议先创建虚拟环境，再安装依赖：

```bash
python -m venv venv
```

### Windows 激活虚拟环境
```bash
venv\Scripts\activate
```

### 安装依赖
```bash
pip install -r requirements.txt
```

---

## 3. 配置运行环境

在运行项目之前，需要根据实际环境配置以下内容：

### Hive 配置
需要配置：
- Hive 主机地址
- 端口
- 用户名
- 数据库名称

### Spark 配置
需要配置：
- 本地模式或集群模式
- Spark Master 地址
- Hive 元数据地址
- HDFS 仓库目录

### Kafka 配置
需要配置：
- Kafka 服务器地址
- Topic 名称
- 消费组名称

### AI 配置
需要配置：
- AI 提供商
- API Key
- Base URL
- 模型名称

---

## 4. 启动相关服务

如果系统依赖完整大数据环境，则需要先启动：

1. Hadoop
2. Hive
3. Spark
4. Kafka

如果只测试部分基础功能，可以在不启用完整大数据组件的情况下运行部分页面，但涉及 Hive / Spark / Kafka / AI 的功能需要对应服务可用。

在项目根目录执行：

python app.py

如果启动成功，终端会显示 Flask 服务运行信息。
